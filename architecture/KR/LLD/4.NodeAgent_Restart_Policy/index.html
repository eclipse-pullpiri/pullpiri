    NodeAgent Self-Healing - Restart Policy 및 Backoff¶

    0. 문서의 목적¶

    • 목표: 종료된 컨테이너를 restart policy와 backoff 알고리즘에 따라 재시작하는 기능
    • 왜 필요한가: 무한 재시작 방지 및 정책에 따른 선택적 재시작이 필요함
    • 이 문서는 NodeAgent의 manager.rs에 handle_exited_container() 함수와 backoff 로직을 추가하기 위해 작성되었습니다.
    • 이 기능은 이 문서에 포함된 조건 및 규칙들을 따라야 합니다.
    • 현재 상태: Restart Policy는 구현되었으나, Backoff는 미구현 (TODO)

    1. Restart Policy의 종류¶

    • Always: 항상 재시작
    • OnFailure: exit code가 0이 아닐 때만 재시작
    • Never: 재시작하지 않음

    2. Backoff 알고리즘¶

    • 첫 재시작: 10초 대기
    • 두 번째: 20초 대기
    • 세 번째: 40초 대기
    • 최대: 300초(5분) 대기
    • 5분(300초) 초과 시: 재시작 시도 중단 (차량 환경 안전)
    • 공식: min(10 * 2^(restart_count), 300) 초

    3. Handle Exited Container를 위해 Manager에 구현되어야 하는 것¶

    ``` +-------------------+ +---------------------+ +-------------------+ | MemoryCache | read | NodeAgent | restart | Podman | | (desired_state) | <------ | (handle_exited_ | ------> | (컨테이너 재시작) | | | | container) | | | +-------------------+ +---------------------+ +-------------------+ ^ | | | backoff_state 메모리 관리 +------------------------------+ ```

    • 인터페이스: 메모리 캐시 조회, Podman API 호출
    • 입력:
      1. 종료된 컨테이너 정보 (DesiredState) 및 exit code
      2. desired_state에서 restart_policy 확인
      3. 메모리에서 backoff_state 조회 (재시작 횟수, 마지막 재시작 시간)
    • 처리:
      1. Restart policy 확인
      2. Never: 즉시 종료
      3. OnFailure: exit_code == 0이면 종료
      4. Always: 계속 진행
      5. Backoff 시간 계산 및 확인
      6. 마지막 재시작 이후 충분한 시간이 경과했는지 확인
      7. 경과하지 않았으면 종료 (다음 루프에서 재시도)
      8. 5분(300초) 이상 경과: 재시작 시도 중단 (차량 안전)
      9. 컨테이너 재시작 (podman restart API)
      10. BackoffState 업데이트 (재시작 횟수++, 마지막 재시작 시간 업데이트)
    • 출력:
      • 컨테이너 재시작
      • backoff_state 메모리 업데이트

    현재 구현 상태: - ✅ Restart Policy (Always/OnFailure/Never) 구현 완료 - ❌ Backoff 알고리즘 미구현 → 1초마다 무한 재시작 발생 가능

    4. 지켜야 할 규칙¶

    • Restart Policy 타입: Always, OnFailure, Never 세 가지 지원
    • `RestartPolicy::Always`: 모든 경우 재시작
    • `RestartPolicy::OnFailure`: exit_code != 0일 때만 재시작
    • `RestartPolicy::Never`: 재시작하지 않음
    • Backoff 계산:
    • 공식: `min(10 * 2^(restart_count), 300)` 초
    • 첫 재시작: 10초, 두 번째: 20초, 세 번째: 40초, ..., 최대: 300초(5분)
    • 5분 초과 시 처리:
    • 마지막 재시작으로부터 300초(5분) 이상 경과 시 재시작 시도 중단
    • 무한 재시작 방지 및 차량 환경 안전 보장
    • 상태 관리:
    • `backoff_states`: 메모리에 HashMap으로 관리 (`HashMap`)
    • `BackoffState`: `{ restart_count: u32, last_restart_time: Option }`
    • 에러 처리: 재시작 실패 시 로그 남기고 backoff_state는 업데이트하지 않음 (다음 루프에서 재시도)
    • 메모리 정리: 컨테이너 삭제 시 backoff_state 제거 (메모리 누수 방지)

    5. 구현 로직 (의사코드)¶

    handle_exited_container 흐름¶

    ``` 입력: desired_state, exit_code

    1. Restart Policy 확인: IF restart_policy == Never: 로그: "RestartPolicy::Never, not restarting" RETURN

    IF restart_policy == OnFailure AND exit_code == 0: 로그: "Exited successfully (code 0), not restarting" RETURN

    // Always 또는 OnFailure with exit_code != 0 계속 진행

    1. BackoffState 조회 (메모리): backoff_state = backoff_states.get(container_id) IF backoff_state 없음: backoff_state = BackoffState { restart_count: 0, last_restart_time: None }

    2. Backoff 시간 확인: IF last_restart_time 있음: elapsed = 현재시각 - last_restart_time

      // 5분(300초) 초과 시 재시작 시도 중단 IF elapsed >= 300초: 로그: "Backoff period expired (5min), stop restarting for vehicle safety" RETURN

      // Backoff 계산 required_backoff = min(10 * 2^(restart_count), 300) 초

      IF elapsed < required_backoff: 로그: "Waiting backoff: {required_backoff - elapsed}초 남음" RETURN // 다음 루프에서 재시도

    3. 컨테이너 재시작: 로그: "Restarting container (attempt #restart_count)"

    result = podman restart container_id

    IF 성공: backoff_state.restart_count++ backoff_state.last_restart_time = 현재시각 backoff_states.insert(container_id, backoff_state) 로그: "Container restarted successfully" ELSE: 로그: "Failed to restart container" // backoff_state 업데이트 안 함 (다음 루프에서 재시도) ```

    Backoff 계산 로직¶

    ```rust fn calculate_backoff(restart_count: u32) -> Duration { let base = 10; // 10초 let wait_seconds = std::cmp::min(base * 2_u64.pow(restart_count), 300); Duration::from_secs(wait_seconds) }

    fn should_stop_restart(last_restart_time: SystemTime) -> bool { let elapsed = SystemTime::now() .duration_since(last_restart_time) .unwrap_or(Duration::from_secs(0)); elapsed.as_secs() >= 300 // 5분 경과 시 재시작 중단 } ```

    현재 구현 (Backoff 없음)¶

    ```rust // manager.rs - reconciliation_loop()에서 호출 async fn handle_exited_container(desired: &DesiredState, exit_code: i32) { use crate::desired_state::RestartPolicy; use hyper::Body;

    let should_restart = match desired.restart_policy {
    RestartPolicy::Always => true,
    RestartPolicy::OnFailure => exit_code != 0,
    RestartPolicy::Never => false,
};

println!(
    "[Reconciliation] Pod '{}': container '{}' exited (code {}), policy={:?}, restart={}",
    desired.pod_name, desired.container_id, exit_code, desired.restart_policy, should_restart
);

if should_restart {
    let restart_path = format!("/v4.0.0/libpod/containers/{}/restart", desired.container_id);
    match crate::runtime::podman::post(&restart_path, Body::empty()).await {
        Ok(_) => println!(
            "[Reconciliation] Container '{}' restarted successfully",
            desired.container_id
        ),
        Err(e) => eprintln!(
            "[Reconciliation] Failed to restart container '{}': {:?}",
            desired.container_id, e
        ),
    }
}

    } ```

    문제점: 1초마다 재시작 시도 → 무한 재시작 루프 발생 (예: helloworld 컨테이너)

    구현 시 주의사항¶

    • Podman API로 컨테이너 재시작: `POST /v4.0.0/libpod/containers/{id}/restart`
    • backoff_states는 컨테이너 삭제 시 정리 필요 (메모리 누수 방지)
    • reconciliation_loop에서 handle_exited_container 호출 시 exit_code 전달 필요
    • 재시작 실패 시 backoff_state를 업데이트하지 않아야 다음 루프에서 재시도 가능
    • 5분 경과 후 재시작 중단으로 차량 환경에서 무한 재시작 방지

    6. Backoff 예시¶

    재시작 시도 restart_count 대기 시간 계산식 비고
    첫 번째 0 10초 10 * 2^0 = 10
    두 번째 1 20초 10 * 2^1 = 20
    세 번째 2 40초 10 * 2^2 = 40
    네 번째 3 80초 10 * 2^3 = 80
    다섯 번째 4 160초 10 * 2^4 = 160
    여섯 번째 5 300초 min(10 * 2^5, 300) = 300 최대값 도달
    5분 경과 후 재시작 중단 무한 대기 elapsed >= 300초 차량 환경 안전
